Description

The TODO app helps you keep track of your activity and improve your personal life. In this project, we will focus on creating an API using the Django REST Framework to add tasks and manage our application.

Before creating an actual API, it is important to understand how Django and Django REST Framework communicate with databases and the outcome they produce. In this stage, you will focus on creating a simple TODO app that will serve as a foundation. You don't need to add fancy templates, user permission, and authorizations. Create an app with Todo model, that has a title, description, and two fields with dates — they show when the task was added and a deadline. Another boolean field also checks whether the task has been completed or not.

Use the Django superuser functionality to add three tasks and a generic view class to display them. Your project should route two URLs: localhost:8000/ and localhost:8000/<id>/.

Objectives

In this stage, your program should:

  • Display an unordered list of all tasks at localhost/8000/. For each task, show its title, date of creation, deadline date, and its status (see the Examples section);
  • Display the detailed view of the task according to the task's ID at localhost/8000/<id>. Here, you should also show the task's description.

You don't have to explicitly define the ID parameter in your model, as it is automatically added by Django.

  • Your Todo model should contain task, ⁣description, goal_set_date, set_to_complete, and is_completed fields.
  • The task contains a task name with a limit of 50 symbols. As for the description field, the max length is 1024.
  • The goal_set_date and set_to_complete fields must contain a date (it should be datetime fields). The is_completed field is boolean.
  • Define Todo in the models.py module and migrate it to the database.

Please, use these exact names for the model and the fields, otherwise your solution might not pass the tests.
